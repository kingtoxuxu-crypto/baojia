# 第一步：定义两种模式的字段边界

## 1. 文档目标

本文件用于明确 `极速模式` 与 `完整模式` 的字段边界、页面边界、结果边界和角色边界，避免后续出现以下问题：

- 极速模式越做越重，失去快速响应能力
- 完整模式沿用极速逻辑，导致正式核价不够深
- 两种模式字段混用，后续数据无法治理

---

## 2. 核心定义

### 2.1 极速模式定义

极速模式是 `经验报价模式`，目标是：

- 在信息不完整时快速给客户一个可谈的价格
- 基于产品模板、历史报价、市场经验、标准参数完成估算
- 输出 `参考价`、`区间价`、`风险提示` 和 `可信度`

极速模式不负责：

- 最终采购落地
- 正式审批定价
- 详细 BOM 成本拆解
- 精确物流和认证结算

### 2.2 完整模式定义

完整模式是 `正式核价模式`，目标是：

- 形成对内可审批、对外可正式发送的报价依据
- 完整记录成本结构、风险项、利润项和参数来源
- 作为订单、采购、生产和财务的前置数据来源

完整模式必须具备：

- 全成本拆解
- 审批口径
- 版本管理
- 参数追溯
- 可冻结结果

---

## 3. 角色边界

### 3.1 极速模式主要使用人

- 销售
- 外贸业务
- 跟单
- 老板 / 总经理

### 3.2 完整模式主要使用人

- 核价员
- 采购
- 工厂管理
- 财务
- 审批人

---

## 4. 字段分类原则

所有报价字段统一分为四类：

### A 类：极速模式必填字段

特点：

- 对价格影响最大
- 客户询价时通常最先知道
- 可以快速录入

### B 类：极速模式自动带出字段

特点：

- 不要求手工录入
- 由模板、经验库、价格库、历史数据自动推导

### C 类：完整模式补录字段

特点：

- 影响正式核价与审批
- 在极速阶段通常拿不到
- 需要后续由核价、采购、工厂补齐

### D 类：系统内部治理字段

特点：

- 面向流程和追溯
- 用户通常不主动填写
- 由系统自动维护

---

## 5. 字段边界矩阵

## 5.1 客户与交易维度

| 字段 | 极速模式 | 完整模式 | 说明 |
| --- | --- | --- | --- |
| 客户名称 | 可选 | 必填 | 极速可先填客户简称 |
| 客户等级 | 必填 | 必填 | 决定利润和策略 |
| 客户区域 | 自动/可选 | 必填 | 可由目标市场带出 |
| 贸易条款 | 必填 | 必填 | EXW/FOB/CFR/CIF/DDP |
| 付款条件 | 不显示 | 必填 | 影响财务成本 |
| 账期天数 | 不显示 | 必填 | 影响资金占用 |
| 佣金政策 | 不显示 | 必填 | 完整模式单列 |
| 返利政策 | 不显示 | 可选 | 大客户场景使用 |
| 售后责任 | 不显示 | 可选 | 影响售后预留 |

| 字段 | 极速模式 | 完整模式 | 说明 |
| --- | --- | --- | --- |
| 产品模板 | 必填 | 必填 | 两种模式共享入口 |
| 产品名称 | 必填 | 必填 | 可默认带出 |
| 型号编号 | 可选 | 必填 | 正式报价需固定 |
| 长宽高 | 必填 | 必填 | 影响包装和物流 |
| 净重 | 必填 | 必填 | 影响陶瓷和物流 |
| 每箱装量 | 必填 | 必填 | 影响包装与柜量 |
| 产品结构参数 | 可选 | 必填 | 如冲水方式、坑距 |
| 是否新品 | 必填 | 必填 | 影响模具与风险 |
| 是否定制 | 可选 | 必填 | 决定审批与成本 |

| 字段 | 极速模式 | 完整模式 | 说明 |
| --- | --- | --- | --- |

## 6. 页面边界

### 6.1 极速模式页面只保留

- 客户与产品
- 市场与条款
- 关键规格
- 包装与数量
- 特殊项标记
- 区间报价结果
- 风险提示

### 6.2 完整模式页面必须具备

- 报价主信息页
- BOM / 成本拆解页
- 采购与供应商页
- 包装与物流页
- 认证与模具页
- 风险与利润页
- 审批与底价页
- 版本与附件页

---

## 7. 输出结果边界

### 7.1 极速模式输出

- 参考单价
- 建议报价区间
- 预计毛利区间
- 风险提示
- 可信度等级
- 转完整模式入口

### 7.2 完整模式输出

- 正式报价单价
- 完全成本
- 毛利额
- 毛利率
- 净利模拟
- 底价
- 审批价
- 审批状态
- 参数来源与版本号

---

## 8. 数据模型边界

建议在报价主表中增加以下字段用于区分模式：

- `quote_mode`
  - `rapid`
  - `full`
- `quote_stage`
  - `estimate`
  - `formal`
- `source_quote_id`
  - 由极速模式转完整模式时记录来源
- `confidence_level`
  - 极速模式可信度
- `pricing_basis`
  - `experience`
  - `historical`
  - `full_cost`

---

## 9. 验收标准

本步骤完成后，应满足以下标准：

1. 所有字段都能被归类到极速、完整或系统字段
2. 极速模式页面中不再出现完整模式专属字段
3. 完整模式不再依赖极速经验字段作为最终依据
4. 两种模式结果名称和口径明确区分
5. 数据库和前端状态层都能识别报价所属模式

---

## 10. 实施顺序建议

1. 先整理字段清单
2. 再整理页面字段映射
3. 再梳理前端状态对象
4. 再调整报价单数据模型
5. 最后再动页面和公式

这样可以避免页面已经改了，但字段口径仍然混乱的问题。
